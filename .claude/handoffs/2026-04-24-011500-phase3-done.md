### Key Patterns Discovered (Phase 3)

- **IPC protocol**: newline-delimited JSON over unix socket. `RequestEnvelope` wraps `Request` with `request_id` (auto-generated ULID if omitted) and optional `client_pid`.
- **Response list variants**: must use named fields (`WorkspaceList { items: Vec<_> }` not `WorkspaceList(Vec<_>)`) due to serde's `#[serde(tag = "kind")]` requiring map-like inner types.
- **Trait disambiguation**: `Store` implements `WorkspaceRepo + AgentRepo + EventLog + AliasRepo` — methods like `save()` and `get()` are ambiguous. Use `WorkspaceRepo::save(&mut *store, &ws)` syntax.
- **Config loading**: `RallyConfig::load()` — defaults → file (JSONC) → env vars. No CLI flag layer yet (Phase 5 or later).
- **Autostart**: CLI spawns `rallyd` on connect failure. Polls for socket up to 3 seconds. Child process is detached.
- **DaemonHarness**: creates temp dir, sets `RALLY_DAEMON_SOCKET_PATH` + `XDG_DATA_HOME`, spawns rallyd, kills on Drop.

### Decisions Made This Session

| Decision | Rationale |
|----------|-----------|
| Newline-delimited JSON (not MsgPack) for IPC | Simpler to debug; Phase 3 MVP doesn't need MsgPack performance. Can swap later. |
| Response list variants as named fields | serde `#[serde(tag)]` can't inject tags into arrays — must wrap in struct |
| Removed dead `warn!` in EventBus::subscribe | Freshly created receiver has no lag; advisor flagged as false-positive signal |
| Filed ral-ieu as P1 bug (not fixed immediately) | Transaction fix needs Store API change; correctness issue but not blocking Phase 3 gate |
